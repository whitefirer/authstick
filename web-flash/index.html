验证码精准送达，物理按键审批

打开登录页的瞬间，服务器根据设备 MAC 地址将验证码推送到你的 StickS3 屏幕。 按下正面按钮批准、侧面按钮拒绝——不走语音、不需输入， 代码出现在对的设备上。

传统认证的三大痛点

TTS 泄露

语音念出验证码

语音助手通过 TTS 播报验证码，周围任何人都能听到。配合屏幕显示的验证流程被语音通道彻底打破。

操作繁琐

切窗口手动输入

看到验证码 → 记住 → 切回网页 → 输入。6 位码来回切换三次，流程断裂、易输错。

设备混用

多设备无隔离

多个终端共享同一套认证系统时，验证码出现在谁的屏幕上？没有设备级身份，审批来源不可追溯。

三步完成认证

网页生成验证码

用户打开登录页面，AuthStick 服务端生成 4 位数字验证码并存入待审批池。

Stick 轮询拉取

StickS3 每 3 秒轮询服务端，带上自己的 MAC 地址。服务端返回当前待审批的验证码到对应设备屏幕。

按键审批 · 即时跳转

正面按键批准 · 侧面按键拒绝。按下瞬间服务端签发 session，网页自动跳转登录。

不止于审批

🛡️

设备注册机制

首次使用需在管理后台输入 6 位验证码绑定设备 MAC，未注册设备无法审批，物理地址天然防冒用。

📡

WiFi 扫码配网

首次开机自动开启热点，手机连接即可在网页中配置 WiFi 和服务器地址。无需编译、无需串口。

⚡

实时倒计时

验证码 5 分钟有效，屏幕实时倒计时。超时自动刷新，过期码无法审批。

🔌

独立认证服务

内置完整 HTTP 服务器，不依赖任何第三方。任何网页接入 3 个 API 即可对接认证。

在线烧录

浏览器一键烧录
无需安装任何工具

使用 Web Serial API 通过 USB 直接烧录固件到 StickS3。仅需 Chrome 或 Edge 浏览器。

  1. 用 USB-C 连接 StickS3 和电脑。
  2. 如果连接失败，按住侧边复位键直到内部绿色 LED 闪烁，再松开。
  3. 点击开始烧录，选择 StickS3 的 USB 串口。
