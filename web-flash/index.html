M5Stack StickS3 · ESP32-S3

物理隔离的硬件认证
设备屏幕显示验证码，网页输入即通过

StickS3 持续显示 6 位验证码，打开登录页直接输入即可。 设备独立令牌加密通信，验证码仅出现在你的设备屏幕上——物理隔离，不可远程截获。

传统认证的三大痛点

通道风险

短信/邮件被截获

验证码走公共通道——短信延迟、邮件泄露、SIM 卡克隆。通道不可控，代码安全性无法保证。

同机认证

TOTP 与登录同设备

手机运行 Authenticator App，登录也在手机上——"你拥有的东西"和"你操作的东西"变成同一台机器，二因子形同虚设。

缺乏身份

硬件 Token 无设备标识

通用 FIDO Key 只证明"我持有密钥"，不能区分是哪把 Key。多设备环境下无法追踪、无法封禁、无法审计。

三步完成认证

StickS3 主动获取验证码

设备定时向服务端请求 6 位验证码并显示在屏幕上，5 分钟有效，过期自动刷新。登录页无需创建，设备即入口。

打开登录页面

在任意设备上打开登录页面，无需注册账号、无需邮箱——页面只有一个输入框。

输入验证码 · 即时通过

将 StickS3 屏幕上的验证码输入网页，提交验证。设备即身份，物理隔离，无法远程截获。

核心特性

🔑

设备令牌 · 不可伪造

每台设备独立生成加密令牌，注册时通过一次性注册码绑定。令牌绝不出现在任何服务端响应中，无法通过伪装MAC地址获取令牌。

📡

热点配网 · 零代码

首次开机自动创建 AuthStick 热点，手机连接后在 192.168.4.1 配置 WiFi 和认证服务地址。无需装驱动、不用连串口。

⏱️

主动获取 · 自动刷新

设备定时向服务端获取 6 位验证码，5 分钟有效，显示精确过期时刻。到期自动刷新，无需任何操作。

🛡️

管理后台 · 全链路可控

设备命名、封禁、移除、重置令牌——管理员拥有完整控制权。被封设备立即失效，令牌可随时重置。

在线烧录

浏览器一键烧录
无需安装任何工具

使用 Web Serial API 通过 USB 直接烧录固件到 StickS3。仅需 Chrome 或 Edge 浏览器。

  1. 用 USB-C 连接 StickS3 和电脑。
  2. 如果连接失败，按住侧边复位键直到内部绿色 LED 闪烁，再松开。
  3. 点击开始烧录，选择 StickS3 的 USB 串口。
